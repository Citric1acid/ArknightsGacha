这是玩家自制的“明日方舟”的模拟寻访系统. 运行方式：
python server.py
有五星保底、六星保底、特定干员概率提升等，与目前游戏内一致.
可以在“添加干员”与“添加卡池”页面添加或更新干员和卡池信息, 例如可以添加自选卡池.
目前添加干员图片仍存在bug, 需要手动添加.

图片等资源来自萌娘百科(https://zh.moegirl.org.cn/明日方舟)和PRTS wiki(https://prts.wiki)
制作者: 柠檬酸, 联系方式 3078614574@qq.com. 欢迎报告bug及提出建议.

v1.2更新：
1. 设置了离线版本和在线版本. 离线版本包含图片，文件较大，在线版本使用网络图片，文件小
2. 添加了“中坚寻访”，新建卡池时可以选择卡池类型，即标准寻访和中坚寻访卡池
3. 添加了单六星150抽大保底机制(限定为120抽)，与游戏中一致
4. 添加卡池时增加了干员名字的自动补全
5. 添加了自动获取干员图片链接的功能(注：暂时无法使用）
6. 启用了python虚拟环境，需要的package已包含
7. 已有干员的显示添加了图片并调整了顺序

v1.0更新：
1. 设置了本地服务器server.py
2. 增添了add.html和add_banner.html, 可以方便地更新信息
3. 更新的信息由服务器直接写入文件
4. 优化了卡池显示的顺序，将最新的显示在第一个

v0.8更新：
（谁说版本号一定要连续的）
1. 保存了卡池信息和已获得干员信息，切换卡池或刷新页面不会重置了；
2. 增加了清除信息按钮；
3. 增加了半手动更新卡池信息的方法；
4. 在更新干员信息时可以批量获取干员图片，虽然由于萌娘百科的防爬虫机制，如果连续运行多次可能被防；
5. 增加了按钮的lock, 避免点击过快时出现bug, 现在必须等上一次寻访的结果完全显示后才可继续点击；
6. 增加了一个卡池中抽卡次数的统计；
7. 优化了页面显示.

v0.5更新：
目前已实现五星保底、六星保底、特定干员概率提升等功能, 更多功能完善中.
现在只有少数干员的图片, 所以大部分干员都无法显示图片, 希望以后能有批量录入的方法.
